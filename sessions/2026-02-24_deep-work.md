## What Worked

Preserve these approaches.

- **Evidence JSON as policy posture snapshot**: Translating the policy library index (pass/fail per control) into Corsair's JSON evidence format produced an immediately auditable baseline. The format is simple enough to maintain by hand or generate programmatically.
- **Sign-then-verify workflow**: Creating the CPOE and immediately verifying it confirmed the cryptographic chain is intact. This two-step process should become standard for every evidence package.
- **Testing all commands before committing to a workflow**: Running cert and tprm exposed their limitations early, before building processes around them.

---

## What Didn't Work

Document for future reference.

- **Homebrew install for Corsair**: `brew install grcorsair/corsair/corsair` is HEAD-only and the formula has a build error (`Errno::ENOENT: No such file or directory - examples`). npm install worked cleanly. Report to grcorsair or revisit when formula is fixed.
- **Expecting cert and tprm to persist state**: Both commands are stateless in v0.6.0 -- vendor registrations and certification state are ephemeral. Don't build workflows around `corsair cert check` or `corsair tprm dashboard` until persistence is added.
- **Feeding markdown files to corsair audit**: Corsair expects JSON evidence files, not markdown. Policy drafts must be translated into evidence JSON before auditing.

## Key Learnings

*Generalizable insights -- candidate entries for memory/learnings.md*

1. **Corsair's core value is the sign/verify/diff loop, not cert/tprm management**: At v0.6.0, the file-based primitives (sign, verify, diff, audit) are production-ready because they operate on files that persist. The higher-level features (cert, tprm) are stateless and not yet operationally useful. Focus on building a library of signed CPOEs as evidence artifacts.
   - Applies when: Evaluating any early-stage GRC tool for operational adoption
   - Evidence: cert create, tprm register, and tprm dashboard all lost state between invocations
   - Confidence: 9/10

2. **Build evidence JSON files as the bridge between policy status and cryptographic verification**: Corsair can't read markdown policies. The evidence JSON format (metadata + controls array with id/description/status/evidence/framework/controlId) is the translation layer. Maintaining these files alongside the policy library index creates a machine-verifiable shadow of your policy posture.
   - Applies when: Using Corsair or any evidence verification tool against a document-based policy library
   - Evidence: Manual evidence JSON with 19 controls produced a full audit, signed CPOE, and governance score in under 5 minutes
   - Confidence: 8/10

3. **Day 0 signed baselines make improvement provable**: The first CPOE at 58% pass / Grade D is valuable precisely because it's low. Every future evidence package can be diff'd against this baseline to show concrete, cryptographically verifiable improvement. For A-LIGN fieldwork, a timeline of signed evidence snapshots is more compelling than a single point-in-time assessment.
   - Applies when: Starting any compliance program or audit prep where posture will improve over time
   - Evidence: Corsair diff command exists specifically for this use case; current CPOE is the anchor point
   - Confidence: 9/10

## Domain Insights

*Anything specific to this system, vendor, framework, or process*

- Corsair v0.6.0 has 13 commands: init, sign, verify, diff, log, audit, cert, tprm, renew, signal, keygen, trust-txt, mappings
- CPOEs are W3C Verifiable Credentials encoded as JWTs, signed with Ed25519 -- any JWT library can verify them
- Corsair scoring has 7 dimensions: sourceIndependence (20%), recency (15%), coverage (15%), reproducibility (15%), consistency (10%), evidenceQuality (15%), completeness (10%)
- Self-assessed document evidence scores 30/100 on sourceIndependence and 40/100 on reproducibility -- the quickest way to raise the score is to add tool-generated evidence (scanner output)
- The `--governance` flag on audit adds a governance score (separate from the summary score) that factors in evidence quality dimensions
- Homebrew formula for Corsair is broken in v0.6.0 (`HEAD-only`, build fails on missing `examples` directory) -- use npm install instead
